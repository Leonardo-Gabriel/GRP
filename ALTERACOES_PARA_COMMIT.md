# Alterações sugeridas para novos commits

Este arquivo organiza os ajustes implementados para diferenciar a versão atual do projeto e permitir uma sequência de commits mais clara.

## Ajustes no código

- Reestruturação do front-end em React com Vite.
- Criação do componente principal em `src/main.jsx`.
- Criação do arquivo de estilos em `src/styles.css`.
- Implementação de indicadores no dashboard.
- Cadastro de culturas agrícolas com validação de campos.
- Listagem em tabela com busca por cultura, tipo ou propriedade.
- Filtro por status da cultura.
- Edição de registros cadastrados.
- Exclusão com confirmação.
- Persistência dos dados no navegador usando `localStorage`.
- Restauração de dados de exemplo.
- Layout responsivo para telas menores.

## Ordem recomendada para commits

1. `Criação da estrutura do front-end com React e Vite`
2. `Implementação do dashboard e indicadores do GRP`
3. `Implementação do cadastro de culturas agrícolas`
4. `Implementação da listagem e filtros de culturas`
5. `Implementação da edição e exclusão de culturas`
6. `Ajustes visuais e responsividade da interface`
7. `Atualização da documentação e evidências da entrega`

## Prints recomendados

- Tela inicial com dashboard e indicadores.
- Formulário de cadastro preenchido.
- Mensagem de cultura cadastrada com sucesso.
- Listagem das culturas cadastradas.
- Filtro ou busca funcionando.
- Tela em modo de edição.
- Confirmação de exclusão.
- GitHub mostrando histórico de commits.
- Terminal com `git log --oneline`.
- Terminal com `git status` limpo.
